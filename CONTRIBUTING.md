# Contributing Guide

Thank you for your interest in contributing to this project!

## Reporting Issues
- Use the **Issues** tab to report bugs or request features.
- Please include clear steps to reproduce, expected behavior, and environment details (app version, OS).

## Development Workflow
- Fork the repository and create a new branch for your changes.
- Write descriptive commit messages.
- Keep pull requests focused on a single change or fix.

## Coding Style
- Follow the existing code style and conventions in the project.
- Use Xcode’s built-in Swift formatter (Editor → Structure → Format with 'swift-format') to keep code consistent.

## Milestones & Releases
- Issues are assigned to milestones that correspond to release versions.
- Milestones are used for tracking progress, while detailed changes are documented in **release notes**.

## Notes
- This project is currently maintained as a solo effort, but contributions of any size (including documentation or typo fixes) are welcome.
